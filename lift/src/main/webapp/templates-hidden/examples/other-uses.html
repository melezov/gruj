Using GRUJ as a glorified poor man's download manager

When plain ol' download managers don't cut it, you need the power of Java connector libraries to tap into them weird astral protocol planes.

Take gopher for example :)
gopher://xsbt.org/ *

* xsbt.org is in no way affiliated with Simple Build Tool

What we're going to do now is just use GRUJ to download the jar file, but let's not actually run it, to keep things simple. This is done via the -m switch like it was shown on the "Executing jars" example. 

C:\>java -jar gruj.jar -m gopher://xsbt.org/sbt-launch-0.11.0.jar sbt-11.jar
[info] Starting gruj v0.1.3 with parameters:
[info]   URL: gopher://xsbt.org/sbt-launch-0.11.0.jar
[info]   File: sbt-11.jar
[info]   Main class: [N/A]
[info] Downloading file (gopher://xsbt.org/sbt-launch-0.11.0.jar) ...
[info] Writing file (C:\sbt-11.jar) ...
[info] Will not run main class, exiting!

Using GRUJ to bootstrap … GRUJ!
